# Proposal: 任务列表状态快速筛选 chips (v2.5 / Epic 37)

## 问题
任务列表工具条已有「优先级快速筛选 chips」(v2.0)，但缺少等价的状态筛选入口。`filterStatus` 信号早已声明并接入 `visibleTasks` 过滤逻辑，却无任何 UI 驱动——用户只能依赖高级筛选面板，操作成本高。

## 目标
在工具条优先级 chips 之后，新增一行「状态快速筛选 chips」：
- 全部 + 6 个状态（待规划/待办/进行中/评审中/验证中/完成），带实时计数与状态色点
- 单击单选切换，再次点击同状态即取消（与优先级 chips 行为对称）
- `localStorage` 持久化选择
- 与「清除全部筛选」按钮联动

## 非目标
- 不改动后端契约（纯前端）
- 不做多选（状态筛选保持单选，与优先级 chips 保持一致体验）

## 风险
低。复用既有 `filterStatus` 信号与 `visibleTasks` 过滤分支，无新后端逻辑；仅新增 UI、计数 computed、持久化与清除联动。
