# Proposal: 任务列表截止日期快速筛选 chips (v2.8 / Epic 40)

## 问题
任务列表工具条已有「优先级 / 状态 / 类型 / 指派人」四组快速筛选 chips（v2.0~v2.7），
但**截止日期（due_date）维度缺失**。代码里 `filterOnlyOverdue` 信号与 `isOverdue()` 逻辑早已存在，
却只在高级筛选面板里有一个孤立的「仅看逾期」勾选框驱动——常驻工具条无等价入口，且无法按
「今天到期 / 本周到期 / 无截止日期」分层筛选，操作成本高、与 chips 家族体验割裂。

## 目标
在工具条指派人 chips 之后，新增一行「截止日期快速筛选 chips」：
- 全部 + 4 个分桶（逾期 / 今天 / 本周 / 无截止），带实时计数
- 单击单选切换，再次点击同分桶即取消（与既有 chips 行为对称）
- `localStorage` 持久化选择（`agentboard_quick_due`）
- 与「清除全部筛选」按钮及高级面板「仅看逾期」开关联动
- 高级筛选面板的「仅看逾期」勾选框复用同一 `filterDueDate('overdue')` 信号，保持单一事实来源

## 非目标
- 不改动后端契约（纯前端，复用既有 `due_date` 字段与 `visibleTasks` 过滤通道）
- 不做多选（保持单选，与现有 chips 一致）
- 不引入「更远日期」分桶（避免工具条过长，超出本周的任务归入「全部」）

## 风险
低。复用 `visibleTasks` 过滤分支与既有 `isOverdue` 日期算法；仅新增分桶方法、计数 computed、
单信号替换原 `filterOnlyOverdue`，并补齐工具条 UI 与样式。原有 overdue 语义（未完成才算逾期）
在分桶内保留。
