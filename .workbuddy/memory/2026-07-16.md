## 踩坑记录

### Docker 网络断连 (重要!)
- **问题**: `docker restart agentboard-api-1` 后容器无网络 (IPAddress 为空, EndpointID 为空)
- **影响**: MCP 容器 (port 18001) 无法解析 `agentboard-api-1` / `api` 主机名 → MCP → API 调用报 `[Errno -2] Name or service not known`
- **解决方案**:
  1. `docker compose stop api && docker compose rm -f api && docker compose up -d --no-deps api` 强制重建才能恢复网络
  2. 重建后立即 `docker exec agentboard-mcp-1 sh -c "echo '<API_IP> api agentboard-api-1' >> /etc/hosts"` (因 MCP 容器禁止重启，只能改 hosts)
- **重要约束**: MCP 容器 (port 18001) **禁止重启/停止**；改用 /etc/hosts 注入 DNS

### 本地代码与容器镜像版本不一致
- **问题**: 本地 `agentboard/models.py` 已重构为 `from .domains.common.enums import ...`，但 Docker 镜像是旧版；单纯 `docker cp` 几个文件导致 `ModuleNotFoundError`
- **解决**: 部署整个 `agentboard` 包到容器，包括 `domains/` 目录；注意 docker cp 目录到已存在目标时会**嵌套复制**，需先 `rm -rf` 旧目录

### MCP 容器与 API 容器网络别名解析
- MCP 环境变量: `AGENTBOARD_API_URL=http://api:8000`（短别名）
- 旧 MCP 容器运行 28h 后 Docker 嵌入 DNS 可能失效
- 最终方案: MCP 容器内 `echo '172.18.0.5 api agentboard-api-1' >> /etc/hosts`
