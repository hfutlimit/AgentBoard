## CORS 头修复（403 显示为 0 Unknown Error）

- 用户截图反馈：jzhong2 访问 `/project/98`（非成员）显示 "Http failure response for http://localhost:18000/api/projects/98: 0 Unknown Error"。
- 根因：FastAPI `CORSMiddleware` 只对经过路由的响应补 CORS 头；`project_access_middleware` 在权限拦截时 `return JSONResponse(...)` 早返回，绕过 CORSMiddleware，浏览器拦截 4xx 后 Angular HttpClient 只看到 status 0。
- 验证：curl `-v` 显示 403 响应缺 `Access-Control-Allow-Origin` 头。
- 修复（commit `f944530`）：新增 `_apply_cors(request, resp)` 辅助函数；`project_access_middleware` 4 处早返回 + `require_business_auth` 1 处全部走 `_apply_cors` 补 CORS 头。
- 验证：重启 api 容器后，403/401 响应均带 `access-control-allow-origin: <origin>` + `access-control-allow-credentials: true` + `vary: Origin`。
- 教训：任何在 middleware 中早返回的 JSONResponse 都必须手动补 CORS 头，否则浏览器会把 4xx/5xx 当成 "0 Unknown Error"，前端无法正常处理错误。
