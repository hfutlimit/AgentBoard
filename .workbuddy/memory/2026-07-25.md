### 可复用模式：SQLite CHECK 约束迁移（重要坑）
SQLite + `PRAGMA foreign_keys=ON` 下，`op.drop_constraint` / `op.create_check_constraint` 报
`NotImplementedError: No support for ALTER of constraints in SQLite dialect`；且 `ALTER TABLE DROP CONSTRAINT ck_xxx`
报 `no such constraint`（命名 CHECK 无法 DROP），`op.batch_alter_table` 反射命名 CHECK 也失败。
**方案**：dialect 分支——非 SQLite 走 `op.drop_constraint`/`op.create_check_constraint`；
SQLite 走**整表重建**（copy-and-move）：用全新 `sqlite3` 连接 `PRAGMA foreign_keys=OFF`，
建新表（内联新 CHECK 约束）→ `INSERT INTO tasks_new SELECT * FROM tasks` → `DROP TABLE tasks` →
`ALTER TABLE tasks_new RENAME TO tasks` → 重建索引；失败回滚删除 `tasks_new`。已验证升级/降级/数据完整。
